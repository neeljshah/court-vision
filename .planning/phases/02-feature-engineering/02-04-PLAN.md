  <name>Task 1: Passing network and momentum modules</name>
  <files>features/passing_network.py, features/momentum.py, tests/test_passing_network.py, tests/test_momentum.py</files>
  <behavior>
    Passing network:
    - Two possession frames where track_id 1 has ball then track_id 2 has ball → PassingEdge(from=1, to=2, count=1)
    - Same pass repeated 3 times in possession → count=3 (edges aggregated, not duplicated)
    - Ball possession determined by closest player to ball position each frame
    - Empty possession → returns []
    - export_network_graph(edges) returns networkx.DiGraph with edge weights = count

    Momentum:
    - [made_by_A, made_by_A, made_by_A] in same segment → scoring_run=3 (pts for A), possession_streak=3
    - Lead change between segments → swing_point=True in the segment where it occurred
    - Empty shot list → returns [] (no segments)
    - segment_size_possessions=5: 12 possessions → ceil(12/5)=3 segments returned
    - scoring_run resets to 0 when a different team scores
  </behavior>
  <action>
    features/passing_network.py:
    - build_passing_network(possession_frames: list[list[dict]], game_id: str, possession_id: str, ball_frames: list[dict | None]) -> list[PassingEdge]
      - possession_frames: list of per-frame player dicts (same format as off_ball_events)
      - ball_frames: parallel list of ball position dicts ({'x', 'y'}) or None if no ball detected
      - Ball holder each frame = player with minimum distance to ball position; if no ball, holder unchanged
      - Track holder transitions: when holder changes from track_id A to B, increment edge (A→B)
      - Aggregate into {(from_id, to_id): count} dict, return list of PassingEdge
    - export_network_graph(edges: list[PassingEdge]) -> networkx.DiGraph
      - Create DiGraph, add_edge(from_track_id, to_track_id, weight=count) for each edge
      - Return the graph (do not persist — caller decides storage)

    features/momentum.py:
    - Shot event format for input: list[dict] with keys 'team' (str), 'made' (bool), 'possession_num' (int), 'timestamp_ms' (float), 'game_id' (str)
    - compute_momentum(shot_events: list[dict], game_id: str, segment_size_possessions: int = 5) -> list[MomentumSnapshot]
    - Group shot_events into segments by possession_num // segment_size_possessions
    - For each segment: sum made shots per team; scoring_run = max consecutive made shots by same team; possession_streak = max consecutive possessions with at least one made shot by same team
    - swing_point = True if the leading team changed between this segment and the previous
    - timestamp_ms = timestamp_ms of last event in segment
    - segment_id = segment index (0-based)
    - Return list sorted by segment_id ascending
  </action>

<task type="auto">
  <name>Task 2: Feature pipeline CLI</name>
  <files>features/feature_pipeline.py, tests/test_feature_pipeline.py</files>
  <action>
    Create features/feature_pipeline.py as a runnable module with argparse CLI.

    Entrypoint: `python -m features.feature_pipeline --game-id <UUID>`

    Pipeline steps (run in order for the given game_id):

    1. Query tracking_coordinates for the game, ordered by frame_number. Separate player rows from ball rows.

    2. Spacing metrics (every 5th frame to manage volume):
       - For each sampled frame, collect player (x, y) positions
       - Call compute_spacing(positions, game_id, possession_id='', frame_number, timestamp_ms)
       - Batch-INSERT results into feature_vectors table (convex_hull_area, avg_inter_player_dist columns)

    3. Defensive pressure (every 5th frame):
       - For each sampled frame, build player_rows dicts from tracking_coordinates
       - Maintain prev_distances dict across frames
       - Call compute_defensive_pressure(player_rows, game_id, frame_number, timestamp_ms, prev_distances)
       - Batch-INSERT results into feature_vectors: UPDATE existing row if frame already written (by spacing), else INSERT with NULL spacing columns

    4. Off-ball events (sliding window of 10 frames, stride 5):
       - For each window, call detect_off_ball_events(window_frames, game_id, ball_pos)
       - INSERT each OffBallEvent into detected_events (event_type, track_id, frame_number, confidence, metadata=NULL)

    5. Pick-and-roll (sliding window of 10 frames, stride 5):
       - For each window, call detect_pick_and_roll(window_frames, game_id)
       - INSERT each PickAndRollEvent into detected_events with event_type='pick_and_roll', metadata=JSON({'screener_track_id': ...})

    6. Passing network (per possession if possessions table has rows for this game, else full game as one possession):
       - Query possessions table; for each possession fetch its frame range from tracking_coordinates
       - Call build_passing_network(possession_frames, game_id, possession_id, ball_frames)
       - Log PassingEdge results to stdout (JSON lines) — no DB table for edges in Phase 2; Phase 5 will consume

    7. Momentum (from shot_logs if rows exist, else skip with warning):
       - Query shot_logs for game_id; build shot_event dicts
       - Call compute_momentum(shot_events, game_id)
       - INSERT each MomentumSnapshot into momentum_snapshots table

    Print summary after each step: step name, records processed, records written.

    Test file (tests/test_feature_pipeline.py):
    - Test that run_feature_pipeline can be imported without error
    - Test that argparse parses --game-id correctly
    - Test step isolation: mock get_connection(), verify each feature module is called
    - Use unittest.mock.patch for all DB calls; do NOT require a live database to run tests
  </action>
