    ```python
    def download_batch(
        clips: Optional[dict] = None,
        max_clips: int = 5,
        max_height: int = 720,
    ) -> List[str]:
        """
        Download multiple clips from CURATED_CLIPS (or a provided dict).

        Skips clips already present in data/videos/ (uses existing download_clip
        skip logic). Stops after max_clips successful downloads.

        Args:
            clips: dict of {label: url/query}. Defaults to CURATED_CLIPS.
            max_clips: Maximum number of clips to download in one batch.
            max_height: Max video height in pixels.

        Returns:
            List of local video paths for all downloaded clips.
        """
        if clips is None:
            clips = CURATED_CLIPS
        paths = []
        for label, url in clips.items():
            if len(paths) >= max_clips:
                break
            try:
                path = download_clip(url, label=label, max_height=max_height)
                if path:
                    paths.append(path)
                    print(f"[download_batch] {label} → {path}")
            except Exception as exc:
                print(f"[download_batch] SKIP {label}: {exc}")
        return paths
    ```

  <name>Task 2: scripts/loop_processor.py — 3-minute recurring pipeline loop</name>
  <files>scripts/loop_processor.py</files>
  <action>
    Create `scripts/loop_processor.py` (new file, max 200 lines). This is a standalone script that runs the full pipeline on any unprocessed clips found in `data/videos/`.

    **Design:**
    - Runs once immediately on launch, then sleeps 180 seconds (3 min) and repeats
    - Low resource usage: single-threaded, no parallel subprocesses, sleep between clips
    - "Unprocessed" = clip file exists in data/videos/ but no corresponding `data/tracking_{clip_stem}.csv` or `data/tracking_data.csv` written for it. Track processed clips in `data/processed_clips.json`.
    - Logs each run to `data/loop_log.jsonl` (one JSON line per clip attempt)

    **Structure:**
    ```python
    """
    loop_processor.py — Recurring NBA clip processing loop.

    Discovers unprocessed clips in data/videos/, runs the full tracking
    pipeline via run_clip.py, and logs results. Repeats every 3 minutes.

    Usage:
        conda run -n basketball_ai python scripts/loop_processor.py
        conda run -n basketball_ai python scripts/loop_processor.py --once  # single pass
        conda run -n basketball_ai python scripts/loop_processor.py --download  # download first
    """
    import argparse
    import json
    import os
    import subprocess
    import sys
    import time
    from datetime import datetime
    from pathlib import Path
    from typing import List

    PROJECT_DIR = Path(__file__).resolve().parent.parent
    VIDEOS_DIR  = PROJECT_DIR / "data" / "videos"
    DATA_DIR    = PROJECT_DIR / "data"
    LOG_FILE    = DATA_DIR / "loop_log.jsonl"
    PROCESSED_FILE = DATA_DIR / "processed_clips.json"
    LOOP_INTERVAL = 180  # 3 minutes
    ```

    **Key functions:**

    1. `load_processed() -> set` — read PROCESSED_FILE, return set of processed filenames. Return empty set if file missing.

    2. `save_processed(processed: set) -> None` — write set to PROCESSED_FILE as JSON list.

    3. `discover_clips() -> List[Path]` — return all `.mp4`, `.mkv`, `.webm` files in VIDEOS_DIR that are not in `load_processed()`. Sort by mtime ascending (oldest first).

    4. `process_clip(clip_path: Path) -> dict` — run the pipeline:
       ```python
       cmd = ["conda", "run", "--no-capture-output", "-n", "basketball_ai",
              "python", str(PROJECT_DIR / "run_clip.py"),
              "--video", str(clip_path), "--frames", "300"]
       result = subprocess.run(cmd, capture_output=True, text=True, timeout=600)
       ```
       Return `{"clip": clip_path.name, "success": result.returncode == 0, "stdout": result.stdout[-500:], "stderr": result.stderr[-200:], "ts": datetime.utcnow().isoformat()}`.
       Timeout after 600 seconds (10 min). On timeout, return success=False.

       NOTE: This command does NOT pass --game-id, so shot_log_enriched will not be
       populated automatically. To populate shot_log_enriched, the caller must add
       `"--game-id", game_id` to the cmd list. See the human verification step for
       how to test with --game-id manually.

    5. `log_result(result: dict) -> None` — append JSON line to LOG_FILE.

    6. `run_loop(once: bool = False, download_first: bool = False)` — main loop:
       - If download_first: call `download_batch(max_clips=5)` from video_fetcher
       - Discover clips → process each → log → mark processed
       - If not `once`: `time.sleep(LOOP_INTERVAL)` then repeat
       - Print `[loop] {datetime.now()} — {len(clips)} clips to process` at start of each pass
       - Print `[loop] sleeping {LOOP_INTERVAL}s...` before sleep

    7. `main()` — argparse: `--once`, `--download`. Call `run_loop()`.

    **Resource usage note:** Do NOT use multiprocessing or threading. Single subprocess per clip, sequential. This is the low-usage design.
